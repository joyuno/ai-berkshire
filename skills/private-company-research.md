# 미상장 기업 리서치: 멀티 Agent 병렬 심층 리서치 프레임워크

$ARGUMENTS 에 대해 팀 기반 심층 리서치 분석을 수행한다. 앤트그룹, 샤오훙수, SpaceX, Stripe 같은 미상장 기업을 위해 설계되었다.

**최종 목표**: 정보가 본질적으로 희소한 조건에서, 이 회사의 **진짜 가치**를 최대한 복원하는 것이다. 시장이 매긴 밸류에이션이 아니라, 사업 그 자체가 얼마짜리인지를 본다.

## 프레임워크 특징

미상장 기업 vs 상장 기업 리서치의 핵심 차이:
- **표준화된 재무제표 없음**: 여러 출처를 짜맞추고 교차 검증해야 한다
- **밸류에이션 기준점 부족**: 투자 라운드, 비교기업법, 시나리오 추론에 의존
- **정보 비대칭 큼**: 더 많은 "퍼즐식" 리서치 방법이 필요
- **엑시트 경로 불확실**: IPO/인수합병/세컨더리 매각 모두 가능

## AI 리서치 편향 자각 (본 프레임워크의 핵심 전제)

미상장 기업은 AI 리서치 편향이 가장 심한 영역이다. 다음 함정을 항상 경계해야 한다:

**핵심 모순**: AI는 이미 존재하는 정보를 구조화하는 데 능하지만, 미상장 기업의 정보는 본질적으로 희소하다. 이로 인해:
1. **거짓 보수성**: 자료가 적으면 AI는 보수적/모호한 결론을 내리는 경향이 있지만, 자료가 적다 ≠ 회사가 나쁘다
2. **거짓 정밀성**: 보고서 템플릿을 채우기 위해 AI가 "합리적 추측"을 "근거 있는 분석"으로 위장할 수 있다
3. **벤치마킹 함정**: 상장 기업과 억지로 비교하면 상장 기업의 밸류에이션 논리를 그대로 물려받아 미상장 기업 고유의 가치를 놓친다
4. **생존자 편향**: 인터넷에서 검색되는 회사 정보는 대개 긍정적으로 편향되어 있다 (회사가 능동적으로 퍼뜨리는 것은 대부분 좋은 소식이다)

**대응 원칙**:
- 추측으로 표를 채워 확실성을 위장하느니, 차라리 "모른다"고 공백으로 남겨라
- 모든 데이터 포인트에 신뢰도를 표기한다 (🟢높음/🟡중간/🔴낮음). 독자가 스스로 판단하게 하라
- "검증 가능한 사실"과 "AI의 추론"을 구분하고, 서로 다른 형식으로 표기한다
- 정보가 극도로 희소한 회사는 "제1원리 모드"로 전환한다. 보고서의 완전성을 추구하지 말고, 다음 몇 가지 핵심 질문에만 답한다:
  1. 이 사업은 어떤 진짜 문제를 해결하는가? 그 수요는 진짜 수요인가, 가짜 수요인가?
  2. 왜 이 팀인가? 그들은 어떤 고유한 강점이 있는가?
  3. 성공한다면 천장은 얼마나 높은가? 실패한다면 가장 가능성 높은 사망 원인은 무엇인가?
  4. 현 단계의 핵심 검증 지점은 무엇인가?

**정보 비대칭의 역이용**: 시장은 미상장 기업에 대한 정보가 적다 → 가격 효율이 낮다 → 바로 그것이 초과수익의 원천일 수 있다. AI 리서치의 목표는 정보 비대칭을 없애는 것(불가능하다)이 아니라, 제한된 정보 안에서 가장 핵심적인 판단 근거를 뽑아내는 것이다.

```
## 재무 데이터 짜맞추기(탐정식 리서치)

### 2. 핵심 재무 지표 추산
다음 데이터를 최대한 추산하되, **모든 데이터 포인트에 반드시 명시**한다: 출처, 시점, 신뢰도, 추산 방법.

**수익 측면**:
- 총수익 규모와 성장률(최근 3년, 연/분기 단위)
- 수익 구조 분해(사업부/제품라인별)
- 수익 성장 동인 분해: 양(사용자/거래량) × 가(ARPU/객단가)
- 수익의 계절성 변동 패턴

**비용 측면**:
- 매출총이익률 추산(동종 상장사 벤치마킹, 선정 논리 설명)
- R&D 비용률 추산(직원 수 × 1인당 급여, 또는 R&D 인력 비중으로 추산)
- 판매 비용률 추산(획득 채널과 집행 데이터로 추산)
- 관리 비용률 추산

**이익 측면**:
- 영업이익/EBITDA 추산
- 순이익/조정 순이익 추산
- 흑자 전환 타임라인: 언제 흑자인가? 아직 적자라면 예상 흑자 시점은?

**현금흐름 측면**:
- 영업현금흐름 판단(양/음, 자체 조혈 가능한가)
- 자본적 지출 수준과 추세
- 잉여현금흐름 추산
- 보유 현금/번레이트(투자 금액, 투자 간격, 직원 수로 추산)
- 현금 런웨이: 현 번레이트로 얼마나 버틸 수 있는가?

**효율 지표**:
- 직원 수와 인당 효율(1인당 수익, 1인당 이익)
- 자본 효율: 1위안 조달당 발생 수익
- 동종 상장사 효율 지표 벤치마킹

### 3. 재무 데이터 교차 검증
- 동일 지표에 복수 출처가 있으면 모두 나열하고 차이를 설명
- 서로 다른 방법으로 같은 지표를 추산하여 결과가 수렴하는지 확인
- 검증 불가능한 "단독 출처" 데이터를 표기

| 지표 | 출처A(데이터/시점) | 출처B(데이터/시점) | 차이 | 채택 판단 |
|------|-------------------|-------------------|------|---------|

### 4. 투자 유치 이력과 밸류에이션 변천
완전한 투자 유치 타임라인을 정리한다:

| 라운드 | 시점 | 금액 | 투자 전 밸류 | 투자 후 밸류 | 리드 투자자 | 공동 투자자 | 밸류 배수 증가 | 비고 |
|------|------|------|---------|---------|--------|--------|------------|------|

분석:
- 밸류에이션 성장 곡선이 건전한가(라운드별 증가 배수가 합리적인가)
- 투자 간격이 합리적인가(너무 잦음=번레이트가 빠름? 너무 드묾=투자 유치 곤란?)
- 밸류에이션 역전(Down Round)이 발생했는가
- 기존 주주가 계속 추가 투자하는가(자신감 신호)
- 최신 라운드의 투자 조건 추측:
  - 우선 청산권(1x/2x/participating)
  - 희석 방지(full ratchet/weighted average)
  - 언아웃/베팅 조항(실적/상장 시점)
  - 이 조항들이 보통주 가치에 미치는 영향

### 5. 밸류에이션 분석(다중 방법 교차)

**방법1: 최근 투자 유치 밸류에이션법**
- 최근 라운드 밸류에이션과 시점
- 우선 청산권 등 조항을 반영한 "보통주 등가 밸류에이션"(보통 20-40% 할인 필요)
- 현재까지의 시점 차이 조정
- 이번 라운드 투자자의 동기 분석(재무적 투자 vs 전략적 투자, 전략 투자자는 프리미엄을 줄 수 있음)

**방법2: 비교 상장기업법**
- 상장 비교기업 3-5개를 선정하고 선정 이유 설명
- 핵심 배수 비교:

| 비교기업 | P/S | P/E | EV/EBITDA | EV/Revenue | 성장률 | 이익률 |
|---------|-----|-----|-----------|------------|------|--------|

- 적용할 조정:
  - 유동성 할인: 20-30%(구체적 수치와 이유 표기)
  - 성장 프리미엄/할인
  - 규모 할인
  - 규제/정책 리스크 할인

**방법3: DCF 시나리오 분석**
세 가지 시나리오, 각각 핵심 가정을 나열한다:

| 가정 | 비관 | 중립 | 낙관 |
|------|------|------|------|

| 종착 영업이익률 | | | |
| 종착 성장률 | | | |
| WACC | | | |
| 종착 배수(EV/EBITDA) | | | |

각 가정은 근거가 뒷받침되어야 하며, 근거 없이 가정해선 안 된다.

**방법4: 종착 시가총액 역산법**
- 5년/10년 후 이 사업이 종착 상태에서 가질 시장 지위 가정
- 종착 수익과 이익률 가정
- 종착 합리 배수(동종 산업 성숙 기업 참고)
- 현재 합리 밸류에이션 구간 역산
- 함의된 연환산 수익률

**방법5: 거래 벤치마크법**
- 동종 산업의 최근 2년 인수합병/투자 유치 거래
- 거래 배수(P/S, P/E)
- 거래 배경과 프리미엄/할인 요인
```

### 4단계: 6개 병렬 Agent 시작

Agent 도구로 6개 Agent를 동시에 시작한다(**반드시 같은 메시지 안에서 병렬 호출**):

각 Agent의 prompt 템플릿:

```
당신은 {회사명} 미상장 기업 리서치 팀의 "{역할 한국어명}"입니다.

당신은 **미상장 기업**을 연구하고 있으며, 이는 다음을 의미합니다:
- 표준화된 공개 재무제표가 없으므로 여러 출처에서 정보를 짜맞춰야 한다
- 데이터가 불완전하거나 서로 모순될 수 있으므로 신뢰도를 표기해야 한다
- 더 많은 추론과 합리적 추산이 필요하지만, 추산 과정을 투명하게 보여야 한다
- 정보를 못 찾음 ≠ 정보가 존재하지 않음, 찾은 정보에는 편향이 있을 수 있다

다음 리서치 태스크를 완료하십시오: {태스크 subject}

구체적 요구사항:
{태스크 description의 내용}

**리서치 방법**:
1. WebSearch로 최신 공개 정보를 검색한다. 각 차원마다 최소 3-5회, 서로 다른 키워드 조합으로 검색
2. 검색 키워드 전략:
   - 중국어: 회사명+수익/밸류에이션/투자유치/사용자수/MAU/IPO/증권신고서/감원/시정
   - 영어: Company Name + revenue/valuation/funding/users/IPO/filing
   - 구체적 인명+회사명(경영진 관련 정보 검색)
   - 회사명+구체적 경쟁자명(경쟁 동태 검색)
3. 중점 정보원:
   - 고신뢰도: 증권신고서, 규제 서류, 상장사 연차보고서 내 관계 공시
   - 중신뢰도: LatePost(晚点), The Information, 36Kr, Bloomberg, Reuters, TechCrunch
   - 보조 검증: 즈후, 마이마이, Glassdoor, 톈옌차, 치차차
4. WebFetch로 핵심 기사의 전문을 가져온다(검색 요약만 보지 말 것)
5. 중요 데이터는 최소 2개의 서로 다른 출처로 교차 검증

**데이터 표기 규범(엄격 준수)**:
- 모든 핵심 데이터에 출처를 표기(구체적 미디어명과 기사 제목까지)
- 데이터 시점을 표기(연월까지 정확히)
- 신뢰도 표기: 🟢높음(증권신고서/공식 공시)/ 🟡중간(신뢰 가능 미디어/리서치)/ 🔴낮음(추산/소문)
- 서로 다른 출처의 데이터가 충돌할 때는 **모두 나열**하고 차이와 당신의 판단을 설명
- "사실"과 "추론"을 구분: 사실은 일반 글꼴, 추론/추산은 *기울임꼴*로 추산 방법을 표기
- 획득 불가능한 정보는 명확히 "데이터 결손"이라 표기, 날조 금지

**산출 요구사항**:
- 보고서는 상세해야 하며, Markdown 표로 핵심 데이터를 제시
- 각 분석 차원마다 명확한 결론과 점수를 제시
- 추산 과정은 완전히 투명해야 함(계산 논리와 각 단계 가정을 보여줄 것)
- 보고서 말미에 제공:
  1. 본 차원의 종합 점수(★1-5)와 핵심 판단
  2. 본 차원 정보 완전도 자가 평가(충분/적정/부족/심각하게 부족)
  3. 가장 중요한 3가지 발견
  4. 가장 큰 정보 사각지대(어떤 정보 결손이 판단에 가장 영향을 주는가)
```

### 6단계: 교차 검증과 정보 퍼즐

**이것이 강화판 프레임워크의 가장 핵심적인 신규 단계다.** 종합 전에 team-lead는 다음을 수행한다:

1. **데이터 충돌 중재**:
   - 각 Agent 보고서의 핵심 데이터를 추출
   - 서로 다른 Agent가 인용한 동일 데이터가 일치하는지 식별
   - 충돌 데이터를 중재: 모든 출처를 나열하고, 어느 것을 채택하는지, 왜 그런지 설명

2. **신호 일관성 검증**:
   - 사업 성장 신호(business-decoder) vs 채용 신호(signal-miner)가 일치하는가?
     (사업이 빠르게 성장한다고 하면서 채용이 축소되고 있다면 설명이 필요)
   - 기술 선도 서사(tech-ip) vs 특허/인재 데이터(signal-miner)가 뒷받침되는가?
   - 밸류에이션 수준(financial-detective) vs 경쟁 지위(competitive-mapper)가 부합하는가?
   - 경영진 공개 서사(risk-governance) vs 실제 행동 신호(signal-miner)가 일치하는가?

3. **정보 퍼즐 복원**:
   - 6건 보고서의 정보 조각을 짜맞춰 더 완전한 그림을 복원할 수 있는지 본다
   - 정보 "백색 지대"(확인된 기지), "회색 지대"(단서는 있으나 불확실), "흑색 지대"(완전 미지)를 표기

4. **반편향 점검**:
   - 보고서에 "긍정 정보는 상세하고 부정 정보는 간략한" 편향이 있는지 점검
   - 모든 긍정 판단에 대응하는 반면 검증이 있는지 확인

## 중요 유의사항

1. **6개 Agent는 반드시 병렬로 시작** — 같은 메시지 안에서 Agent 도구를 6번 호출
2. **데이터 신뢰도 표기** — 미상장 기업은 데이터 출처가 들쭉날쭉하므로, 모든 핵심 데이터에 출처와 신뢰도를 반드시 표기
3. **추산은 투명하게** — 모든 추산 과정은 계산 논리를 보여야 하며, 근거 없이 숫자를 제시할 수 없음
4. **교차 검증** — 핵심 데이터는 최소 2개 출처로 교차 검증, 출처 충돌 시 모두 나열
5. **신호 일관성 검증** — 종합 단계에서 반드시 차원 간 신호 일관성 점검을 수행
6. **결론은 명확하게** — 투자/관망/회피 권고 제시를 회피하지 말되, 동시에 결론의 신뢰도를 설명
7. **인내심 있게 대기** — 6개 Agent 연구에는 몇 분이 걸리니, 사용자에게 실시간으로 진행을 갱신
8. **중영문 검색** — 미상장 기업 정보는 중국어/영어 미디어에 분산되어 있을 수 있으니, 두 언어로 검색
9. **반편향 핵심 원칙** — 자료가 적다 ≠ 회사가 나쁘다, AI 분석 분량이 짧다 ≠ 투자 확실성이 낮다. 정보가 극도로 희소한 회사는 "제1원리 모드"로 전환하여 핵심 질문에 집중하고, 보고서 형식의 완전성을 추구하지 않음
10. **정직한 공백** — 보고서에서 "근거 있는 분석"과 "추측 채움"을 명확히 구분하고, "이 차원은 데이터가 부족하여 의미 있는 결론을 낼 수 없음"이 나오는 것을 허용
11. **대체 데이터는 노이즈가 아니다** — 채용, 특허, 소송, App 데이터 등 대체 데이터는 뉴스 보도보다 실제 경영 상황에 더 가까울 수 있음
12. **진짜 가치 지향** — 최종 목표는 이 사업이 얼마짜리인지 판단하는 것이지, 보기 좋은 보고서를 산출하는 것이 아니다. 정보가 신뢰 가능한 밸류에이션 판단을 내리기에 부족하다면, 직접 "정보 부족, 신뢰 가능한 밸류에이션을 제시할 수 없음"이라고 말하라
