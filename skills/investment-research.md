# 투자 리서치: Warren Buffett(버핏)-Charlie Munger(멍거)-段永平(돤융핑)-李录(리루) 4대가 종합 분석 프레임워크

$ARGUMENTS 에 대해 체계적인 투자 리서치 분석을 수행한다.

## 리서치 프레임워크

Warren Buffett(버핏), Charlie Munger(멍거), 段永平(돤융핑), 李录(리루) 네 명의 투자 대가가 정립한 방법론을 토대로, 아래 일곱 개 모듈을 순서대로 수행해 리서치를 진행한다.

### 사전 단계: AI 리서치 편향 자각(반드시 수행)

리서치를 시작하기 전에, 먼저 해당 기업의 "AI 리서치 가능성"을 평가하고 잠재적 데이터 편향을 식별한다.

**정보 풍부도 등급**:
| 등급 | 특징 | AI 리서치 함정 | 대응 전략 |
|------|------|-----------|---------|
| A급(정보 풍부) | 상장 이력이 길고, 증권사 커버리지가 많으며, 언론 보도가 빈번 | 컨센서스가 지나치게 강해 AI 산출물이 시장 가격에 수렴하고 알파가 제한적 | 반대 검증에 집중: 똑똑한 사람들은 왜 안 사는가? 간과된 리스크는 무엇인가? |
| B급(정보 적정) | 상장 1~3년, 커버리지 제한적, 일부 데이터는 추정 필요 | AI가 "합리적 추측"으로 공백을 메워, 완결돼 보이지만 실제로는 허위 확실성 | 추정 데이터마다 신뢰도를 표기하고, "근거 있는 추정"과 "근거 없는 채워넣기"를 구분 |
| C급(정보 희소) | 상장 직후/비인기주/신흥시장, 커버리지 거의 없음 | AI가 자료 부족으로 과도하게 보수적이 되어 "안 보인다=나쁘다"로 오판 | 제1원리 질문(아래 참조)을 활용해, 제한된 정보에서 비즈니스의 본질을 추출 |

**C급 기업의 제1원리 리서치법**:
공개 자료가 부족할 때는 "완결돼 보이는" 보고서를 억지로 짜맞추려 하지 말고, 아래 근본 질문에 집중한다:
1. 고객은 누구인가? 왜 돈을 내는가? 대체 선택지가 있는가?
2. 재구매를 무엇이 견인하는가? 습관인가, 락인인가, 아니면 지속적인 신규 가치 창출인가?
3. 경쟁자가 100억을 들이면 이 사업을 복제할 수 있는가?
4. 경영진은 어떤 핵심 의사결정을 내려왔는가? 그 결정들은 어떤 판단력과 가치관을 반영하는가?

**편향 자가 점검 체크리스트**(리서치 전 과정 동안 경계 유지):
- [ ] 나의 "확신"은 사업의 본질에서 오는가, 아니면 자료의 양에서 오는가?
- [ ] 이 기업의 자료량을 절반으로 줄이면 내 결론이 바뀌는가?
- [ ] AI가 내놓은 분석이 시장 컨센서스와 지나치게 비슷한가? 그렇다면 나의 정보 우위는 어디에 있는가?
- [ ] "공개 자료는 적지만 사업의 본질은 극히 우수한" 가능성이 과소평가되고 있지는 않은가?

정보 풍부도 등급 평가 결과를 보고서 서두에 기재하고, 최종 결론에서 "AI 리서치 신뢰도"와 "실제 투자 확실성"의 차이를 명시한다.

### 1단계: 데이터 수집

> **데이터 소스 규범**: `skills/financial-data.md` 참조. 모든 재무 데이터는 두 개의 독립된 소스에서 확보해야 하며, 오차가 1%를 초과하면 표기해야 한다.
> - 미국주: macrotrends(주) + stockanalysis(부)
> - 홍콩주: aastocks(주) + macrotrends ADR(부)
> - A주: 동방재부(주) + 거조자신(부); `py tools/ashare_data.py` 사용 가능
> - 한국주: `py tools/krx_data.py`(네이버 금융, 주) + DART/KRX(부)

Task 도구로 백그라운드 Agent를 실행해, 웹에서 아래 데이터를 수집한다:

1. 수익 구조: 최근 회계연도 및 최근 4개 분기의 부문별 매출, 성장률, 매출총이익률
2. 재무 지표: 최근 5년 매출, 순이익, 매출총이익률, 영업이익률, 잉여현금흐름, 현금 보유고
3. 경쟁 구도: 시장 점유율, 주요 경쟁사 비교
4. 비즈니스 모델과 해자: 핵심 경쟁우위의 원천
5. 기술 역량: 핵심 기술 스택, R&D 투자
6. 경영진: 창업자/CEO 이력, 지분율, 핵심 의사결정 기록
7. 산업 전망: TAM(총유효시장), 성장 전망
8. 리스크 요인: 지정학, 규제, 공급망 등
9. 현재 밸류에이션: 시가총액, PE, PS, PEG, EV/Revenue
10. 매수·매도 양측의 핵심 논거

#### 데이터 교차 검증(반드시 수행, 금융 엄밀성 도구 사용)

데이터 수집이 끝나면, **반드시 `tools/financial_rigor.py` 를 호출해 핵심 데이터를 프로그램으로 검증**하여 LLM 암산 오류를 차단한다.

**반드시 검증해야 할 데이터 포인트**:
- 총주식수(거래소, Yahoo Finance, StockAnalysis 등 최소 2개 소스에서 확인)
- 현재 주가와 시가총액(**주가×총주식수를 직접 계산해 보고서 시가총액과 대조하여 단위 오류 방지**)
- 최근 회계연도 매출과 순이익(회사 연차보고서 + 최소 1개 제3자 소스에서 확인)
- 현금 보유고와 순현금(현금+단기투자-총부채, 산정 기준 차이에 유의)
- 경영진 지분율(경제적 지분과 의결권 구분, AB주 구조에 유의)

**필수 검증 단계(Bash로 도구 호출)**:

Step 1 — 시가총액 검산(부동소수점이 아닌 정밀 십진수):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap \
  --price {주가} --shares {총주식수} --reported {보고서 시가총액} --currency {통화}
```

Step 2 — 핵심 데이터 다중 소스 교차 검증:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate \
  --field {필드명} --values '{"소스1": 수치, "소스2": 수치}' --unit {단위}
```
매출, 순이익, 현금 보유고에 대해 각각 수행한다.

Step 3 — 밸류에이션 지표 정밀 검산(PE/PB/ROE/FCF Yield 등):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {주가} --eps {EPS} --bvps {주당순자산} --fcf-per-share {주당 FCF} --dividend {주당 배당금}
```

**검증 규칙**:
1. 모든 핵심 데이터 포인트는 최소 2개의 독립된 소스를 확보한다
2. 소스 간 차이를 발견하면 회사 연차보고서/거래소 데이터를 우선 채택하고, 차이의 원인을 명시한다
3. **계산이 수반되는 모든 데이터는 반드시 도구로 검산해야 하며, LLM 암산을 금지한다**
4. 도구 출력 결과를 보고서 부록 "핵심 데이터 교차 검증 기록"에 그대로 삽입한다
5. 도구가 ❌ 편차 과다를 보고하면, 원인을 규명한 뒤에야 분석을 이어갈 수 있다

**자주 발생하는 오류 예방**:
- 시가총액 단위: 홍콩달러 억 vs 위안화 억 vs 미국달러 억, 0을 빠뜨리거나 더하기 쉬움
- FCF 산정 기준: 소스마다 자본적지출의 정의가 다를 수 있음(리스·인수 등 포함 여부)
- 부채 산정 기준: 운용리스 부채 포함 여부
- 지분율: AB주 기업의 경제적 지분 ≠ 의결권

### 2단계: 사업 본질 분석 — 段永平(돤융핑)의 "옳은 사업"

분석 포인트:
- 이 사업의 본질을 한 문장으로 정의
- 수익 구조 분해(차트)
- 5년 수익성 추세(차트)
- 비즈니스 모델 캔버스: 일회성 판매 vs 구독/재구매? 하드웨어 vs 소프트웨어 vs 플랫폼?
- 생태계 점착성/고객 락인 강도
- 매출총이익률 수준과 동종업계 비교, 왜 높은지/낮은지 설명
- 영업 레버리지 분석
- **段永平(돤융핑)식 추궁**: 이 사업의 좋은 점은 무엇인가? 한 문장으로만 묘사해야 한다면 무엇인가?

### 3단계: 해자 평가 — Warren Buffett(버핏)의 "경제적 해자"

다섯 종류의 해자를 하나씩 검증한다:

| 해자 유형 | 검증 방법 |
|-----------|---------|
| 브랜드/가격 결정력 | 판매량 손실 없이 가격을 올릴 수 있는가? |
| 전환 비용 | 고객이 경쟁 제품으로 이전하는 비용이 얼마나 높은가? |
| 네트워크 효과 | 사용자가 많을수록 제품이 더 좋아지는가? |
| 규모 효과 | 규모가 가져오는 원가 우위는 얼마나 큰가? |
| 기술/특허 장벽 | 기술이 몇 년 앞서 있는가? 복제 가능한가? |

해자 추세 분석: 지난 5년간 넓어졌는가, 좁아졌는가? 향후 5년 전망.

**Warren Buffett(버핏)식 추궁**: 10년 뒤에도 이 해자가 존재하는가? 무엇이 그것을 무너뜨릴 수 있는가?

### 4단계: 역발상과 리스크 목록 — Charlie Munger(멍거)의 "거꾸로 생각하라"

- "이 회사가 실패할 수 있는 모든 경로"를 나열(표: 경로/확률/영향 정도)
- 역사적 유추: 역사상 비슷한 위치에 있던 회사를 찾아, 그 결말은 어땠는가?
- 다학제 분석: 네트워크 효과 이론, 기술 채택 곡선, 경쟁 게임이론 등 모델로 교차 검증
- 편향 자가 점검: 서사 편향, 앵커링 효과, 생존자 편향
- 매도 측의 핵심 논거 수집

**Charlie Munger(멍거)식 추궁**: 나는 어디서 틀릴 가능성이 가장 높은가? 똑똑한 사람들은 왜 이 회사를 안 사거나 공매도하는가?

### 5단계: 경영진 평가 — 段永平(돤융핑)의 "옳은 사람" + Warren Buffett(버핏)의 "경영진 정직성"

- CEO/창업자의 핵심 의사결정 복기(표: 시점/결정/결과/점수)
- 자본 배분 능력: R&D 수익률, 인수합병 성공률, 자사주 매입 타이밍
- 주주 이익 정합성: 경영진 지분, 보수 구조, 지분 매도 기록
- 조직 역량: 팀 안정성, 핵심 인재 리스크
- 기업 문화 특징

**段永平(돤융핑)식 추궁**: CEO가 은퇴해도 이 회사는 경쟁력을 유지할 수 있는가?

### 6단계: 산업과 문명 추세 — 李录(리루)의 "문명 진화 프레임워크"

- 해당 산업이 "문명급 패러다임 전환"에 있는지 판단
- 역사적 기술 혁명 유추(증기기관/전기/인터넷/AI)
- TAM 성장 곡선과 천장 분석
- 산업 가치사슬 내 회사의 위치
- 기술 노선 리스크
- 고객/공급사 집중도 분석

**李录(리루)식 추궁**: 20년 뒤에서 돌아본다면, 이 회사는 "이 시대의 스탠더드 오일"인가, 아니면 "반짝하고 사라진 3Com"인가?

### 7단계: 밸류에이션과 안전마진 — Warren Buffett(버핏)의 "내재가치" + 段永平(돤융핑)의 "옳은 가격"

- 현재 시장의 가격 평가(핵심 밸류에이션 지표 표) —— **반드시 도구로 검산**
- 역산 DCF: 현재 주가는 어떤 성장 기대를 내포하고 있는가?
- 세 가지 시나리오 밸류에이션 —— **반드시 도구로 정밀 계산, 암산 금지**:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario \
  --price {주가} --eps {EPS} --shares {총주식수 억} \
  --growth {낙관 성장률} {중립 성장률} {비관 성장률} \
  --pe {낙관 PE} {중립 PE} {비관 PE} --years 3 --currency {통화}
```
- 자사 과거 밸류에이션과 비교
- 동종업계 밸류에이션과 비교

**段永平(돤융핑)식 추궁**: 내일 주식시장이 5년간 문을 닫는다 해도, 이 가격에 보유하겠는가?

## 출력 요건

1. 모든 분석에는 데이터 근거가 있어야 하며, 데이터 출처를 첨부
2. Markdown 표로 핵심 데이터를 제시
3. 각 모듈 말미에는 해당 대가의 "추궁"이 반드시 있어야 함
4. 최종적으로 완성된 보고서를 `~/[회사명]투자리서치보고서.md` 에 기록
5. 결론은 명확해야 하며, 매수/관망/회피 권고를 회피하지 말 것
6. 밸류에이션 부분은 구체적인 가격 구간을 제시해야 함
7. **보고서 서두**에는 "정보 풍부도 등급"(A/B/C)과 "AI 리서치 한계 선언"을 반드시 포함
8. **보고서 말미**에는 "AI 분석 신뢰도"와 "투자 확실성"을 구분해야 함 —— 전자는 자료량에, 후자는 사업의 본질에 좌우된다. 본 보고서의 어떤 결론이 충분한 데이터에 기반하고, 어떤 결론이 제한된 정보의 추론에 기반하는지 독자에게 명확히 알릴 것
9. 회사가 C급(정보 희소)에 해당하면, 보고서 말미에 "1차 검증이 필요한 질문 목록"을 반드시 나열 —— 독자가 현장 조사, 제품 체험, 공급망 인터뷰 등으로 AI의 사각지대를 보완하도록 권고

## 데이터 표본 점검(출고 절차)

보고서를 파일에 기록한 뒤, **반드시** 데이터 표본 점검을 수행하고, 통과해야만 발행할 수 있다:

**Step 1 — 점검 목록 추출(15% 무작위 표본):**
```bash
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <보고서 파일 경로>
```
JSON 템플릿을 출력하며, 각 항목에 `fetched_value`(입력 대기)가 포함된다.

**Step 2 — 데이터 취득 검증:**
목록의 각 데이터 포인트에 대해, `skills/financial-data.md` 규범에 따라 신뢰할 수 있는 출처에서 데이터를 취득
(미국주: macrotrends+stockanalysis; 홍콩주: aastocks+macrotrends; A주: 동방재부+거조자신; 한국주: krx_data.py/네이버 금융+DART),
`fetched_value` / `fetched_source` / `fetched_value2` / `fetched_source2` 에 입력한다.

**Step 3 — 판정 출력:**
```bash
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<입력 완료된 JSON>' \
  --report <보고서 파일명>
```

- **【출고 승인】**: 모든 점검 포인트의 편차 ≤ 1% → 보고서 발행 가능
- **【반려】**: 어느 한 포인트라도 편차 > 1% → 해당 데이터를 수정한 뒤 재점검, 출고 승인될 때까지 반복
