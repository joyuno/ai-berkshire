# 투자 리서치 팀: 4역할 병렬 분석 프레임워크

$ARGUMENTS 에 대해 팀 기반 투자 리서치 분석을 수행한다. Team 도구를 사용해 실제 멀티 Agent 병렬 리서치 팀을 구성한다.

## 실행 흐름

### 1단계: 팀 프레임워크 제시

사용자에게 아래 팀 구조를 제시하고, 확인을 받은 뒤 시작한다:

| 역할 | 직무 | 분석 프레임워크 |
|------|------|----------|
| **team-lead**(당신 자신) | 총괄 조율, 종합 판단, 최종 보고서 산출 | 4대가 종합 프레임워크 |
| **business-analyst** | 비즈니스 모델 & 해자 분석 | 돤융핑(段永平) 관점 |
| **financial-analyst** | 재무제표 & 밸류에이션 분석 | 버핏 관점 |
| **industry-researcher** | 산업 구도 & 경쟁 양상 | 멍거 관점 |
| **risk-assessor** | 리스크 평가 & 경영진 판단 | 리루 관점 |

### 1.5단계: AI 리서치 편향 평가

팀을 구성하기 전에, 먼저 사용자에게 해당 기업의 "AI 리서치 가능성" 평가를 제시한다:

**정보 풍부도 등급**(리서치 전략을 결정):
| 등급 | 특징 | 리서치 전략 조정 |
|------|------|------------|
| A급(정보 풍부) | 상장 이력이 길고, 증권사 커버리지가 폭넓음 | 팀은 **반대 검증**과 **비컨센서스 관점**에 집중하여, 시장과 일치하는 "옳은 헛소리" 산출을 회피 |
| B급(정보 적정) | 상장한 지 얼마 안 됐고, 커버리지 제한적 | 각 Agent의 추정 데이터는 반드시 신뢰도를 표기하고, team-lead 종합 시 "데이터 충분도"를 표기 |
| C급(정보 희소) | 비인기주/신규 상장/신흥시장 | 팀을 "제1원리 모드"로 전환: 보고서 완결성을 추구하지 않고, 비즈니스 본질의 몇 가지 핵심 질문에 집중 |

**핵심 유의**: 자료가 많다고 확실성이 높은 것이 아니고, 자료가 적다고 확실성이 낮은 것이 아니다. AI가 산출할 수 있는 신뢰도 ≠ 투자의 실제 확실성. 확실성은 비즈니스 모델 자체에서 오는 것이지, 자료의 양에서 오지 않는다.

등급 평가 결과를 각 Agent에게 알려, 그 리서치 방식에 반영되게 한다.

### 2단계: 팀 생성

TeamCreate 로 팀을 생성한다:
- team_name: `{회사명}-research`(영문 소문자, 예: `meituan-research`)
- agent_type: `team-lead`

### 3단계: 4개 태스크 생성

TaskCreate 로 아래 4개 태스크를 생성한다(각각 subject, description, activeForm 을 모두 포함해야 함):

#### 태스크1: 비즈니스 모델 분석
- subject: `{회사명}의 비즈니스 모델, 해자, 사용자 가치 분석`
- description 포함 내용:
  1. 비즈니스 모델 본질: 핵심 사업 정의, 수익 구조 분해
  2. 플랫폼/제품 플라이휠 효과의 작동 방식
  3. 해자 분석: 브랜드/전환 비용/네트워크 효과/규모 효과/기술 장벽을 하나씩 검증
  4. 사용자/고객 가치: 각 주체에게 어떤 독특한 가치를 창출했는가
  5. 사업 매트릭스와 시너지 효과
  6. 돤융핑(段永平)의 "좋은 사업" 기준 평가: 차별화, 가격 결정력, 지속 가능한 경쟁우위
  7. 최신 실적 보고서, 산업 리포트 등 공개 정보 검색 요구

#### 태스크2: 재무 및 밸류에이션 분석
- subject: `{회사명}의 재무 데이터, 수익성, 밸류에이션 분석`
- description 포함 내용:
  1. 최근 3~5년 매출, 순이익, 영업이익 추세
  2. 수익성 지표: ROE, ROA, 매출총이익률, 영업이익률
  3. 현금흐름 분석: 영업현금흐름, 잉여현금흐름, 자본적지출
  4. 재무상태표 건전성: 현금 보유고, 부채비율, 유동성
  5. 밸류에이션 분석: PE/PS/PB/EV 등, 과거 및 동종업계와 비교
  6. 안전마진 평가: 내재가치 vs 현재 주가
  7. **금융 엄밀성 검증(반드시 Bash로 도구 호출, 암산 금지)**:
     - 시가총액 검산: `python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap --price {가격} --shares {주식수} --reported {보고서 시가총액} --currency {통화}`
     - 밸류에이션 검산: `python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation --price {가격} --eps {EPS} --bvps {주당순자산}`
     - 핵심 데이터 교차 검증: `python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate --field {필드} --values '{JSON}' --unit {단위}`
     - 세 가지 시나리오 밸류에이션: `python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario --price {가격} --eps {EPS} --shares {주식수 억} --growth {낙관} {중립} {비관} --pe {낙관PE} {중립PE} {비관PE}`
     - 도구 출력 결과를 보고서에 그대로 삽입해 검증 기록으로 삼는다

#### 태스크3: 산업 및 경쟁 분석
- subject: `{산업}의 산업 구도와 {회사명}의 경쟁 양상 분석`
- description 포함 내용:
  1. 산업 규모와 성장: 시장 규모, 성장률, 침투율
  2. 경쟁 구도: 주요 경쟁사의 시장 점유율, 경쟁 전략 비교
  3. 핵심 경쟁자 위협 평가: 주요 경쟁사를 하나씩 분석
  4. 각 세부 영역의 구도
  5. 산업 추세: 기술 변혁, 정책 영향, 신규 진입자
  6. 가치사슬 분석: 전·중·후방의 가치 분배
  7. 최신 산업 데이터와 경쟁 동향 검색 요구

#### 태스크4: 리스크 및 경영진 평가
- subject: `{회사명}의 투자 리스크와 경영진 품질 평가`
- description 포함 내용:
  1. 경영진 평가: CEO 능력범위, 정직성, 전략적 안목, 자본 배분 능력, 과거 의사결정 품질
  2. 규제 리스크: 현재 및 잠재적 규제 영향
  3. 경쟁 리스크: 각 경쟁사의 위협 정도 평가
  4. 사업 리스크: 신규 사업 적자, 확장의 불확실성
  5. 거시 리스크: 경기 사이클, 산업 사이클 영향
  6. 지배구조: 지분 구조, 특수관계자 거래, 주주 환원 정책
  7. 장기 확실성: 10년 뒤 회사는 어떻게 될 것인가? 무엇이 그 비즈니스 모델을 뒤집을 수 있는가?
  8. 최신 규제 동향, 경영진 발언 등 검색 요구

### 4단계: 4개 병렬 Agent 실행

Task 도구로 4개 Agent를 동시에 실행한다(**반드시 같은 메시지 내에서 병렬 호출**):

각 Agent의 설정:
- `subagent_type`: `general-purpose`
- `run_in_background`: `true`
- `team_name`: 해당 팀명
- `name`: 해당 역할명(business-analyst / financial-analyst / industry-researcher / risk-assessor)

각 Agent의 prompt 템플릿:

```
당신은 {회사명} 투자 리서치 팀의 "{역할 한국어명}"이며, {대가명} 투자 관점에서 {회사명}을 분석하는 담당이다.

태스크 #{태스크 번호}: {태스크 subject} 를 완료하라.

구체적 요구사항:
{태스크 description의 내용}

**리서치 방법**:
- WebSearch 로 최신 공개 정보(실적 보고서, 산업 리포트, 뉴스)를 검색
- **재무 데이터는 반드시 두 개의 독립된 소스에서** 확보하고, `skills/financial-data.md` 규범에 따라 수행(미국주: macrotrends+stockanalysis; 홍콩주: aastocks+macrotrends; A주: 동방재부+거조자신), 두 소스 오차>1%는 표기
- 데이터의 정확성을 보장하고, 핵심 데이터는 출처를 표기
- 분석은 깊이 있게, 표면에 머무르지 말 것

**출력 요건**:
- 보고서는 상세해야 하며, Markdown 표로 핵심 데이터를 제시
- 각 분석 차원마다 명확한 결론과 점수가 있어야 함
- 보고서 말미에는 해당 차원의 종합 결론이 있어야 함

**완료 후**:
1. TaskUpdate 로 태스크 #{태스크 번호} 를 completed 로 표시
2. SendMessage 로 완성된 분석 보고서를 team-lead 에게 전송(type: "message", recipient: "team-lead")
```

### 5단계: 보고서 수신 및 진행 상황 추적

- 사용자에게 진행 표를 실시간으로 제시(어떤 Agent가 완료됐고, 어떤 Agent가 아직 리서치 중인지)
- 보고서를 한 건 받을 때마다 진행 상황을 갱신하고, 그 보고서의 핵심 요점(3~5개)을 제시
- 4건의 보고서가 모두 도착할 때까지 대기

### 6단계: 팀원 종료

모든 보고서를 받은 뒤, 4개 Agent에게 shutdown_request 를 전송한다(SendMessage 사용, type: "shutdown_request").

### 7단계: 최종 보고서 종합

4건의 분석 보고서를 종합하여, 아래 구조의 최종 보고서를 산출한다:

---

#### 1. 한 문장 결론
> 한 단락(50~100자)으로 투자 가치 여부와 핵심 논리를 요약

#### 2. 4차원 점수 총괄표
| 차원 | 프레임워크 | 점수(1-5★) | 핵심 판단 |
|------|------|------------|----------|

종합 점수: X / 5

#### 3. 핵심 데이터 한눈에 보기
핵심 재무·경영 지표 표(최근 2년 비교)

#### 4. 각 차원 분석 요약
각 차원에서 가장 중요한 발견 3~5개를 발췌

#### 5. 투자 논점(Bull vs Bear)
- 🟢 매수 논리(5~7개)
- 🔴 매도 논리(5~7개)

#### 6. 버핏의 매수 전 Checklist
| # | 점검 항목 | 통과? | 설명 |
10개 핵심 점검 항목을 하나씩 평가

#### 7. 최종 투자 권고
- 정성 판단 표(사업 품질/경영진/밸류에이션/타이밍)
- 계층별 매매 권고 표(공격형/안정형/보수형 → 권고+가격 구간)
- 핵심 촉매(추가 매수 시그널/축소 시그널 각 3~5개)

#### 8. 결론 단락
100~200자의 최종 결론

---

### 8단계: 보고서 저장

완성된 최종 보고서를 `~/{회사명}투자리서치보고서_{날짜}.md` 에 기록(날짜 형식 YYYYMMDD).

### 9단계: 데이터 표본 점검(출고 절차)

```bash
# Step 1 — 점검 목록 추출(15% 무작위 표본)
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <보고서 파일 경로>

# Step 2 — 목록의 각 항목을 신뢰할 수 있는 출처에서 취득(skills/financial-data.md 참조)

# Step 3 — 출고 승인/반려 판정 출력
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<입력 완료된 JSON>' \
  --report <보고서 파일명>
```

**【출고 승인】** 전부 통과 → 보고서 발행 가능; **【반려】** 미통과 항목 존재 → 수정 후 재심사.

### 10단계: 팀 정리

TeamDelete 로 팀 리소스를 정리한다.

## 중요 유의사항

1. **4개 Agent는 반드시 병렬로 실행** —— 같은 메시지 내에서 Task 도구를 4번 호출
2. **Agent는 SendMessage 로 보고** —— 파일 협업이 아니라 메시지 통신
3. **데이터 정확성** —— Agent에게 WebSearch 로 최신 데이터를 검색하도록 요구하고, 핵심 데이터는 교차 검증
4. **결론은 명확하게** —— 매수/관망/회피 권고와 구체적 가격 구간 제시를 회피하지 말 것
5. **모든 분석에는 데이터 근거가 있어야 함** —— 데이터 출처 첨부
6. **인내심 있게 대기** —— 4개 Agent의 리서치에는 몇 분이 걸리므로, 사용자에게 진행 상황을 실시간으로 갱신
7. **반편향 의식** —— team-lead 는 종합 시 반드시 평가할 것: 각 Agent의 분석이 자료 풍부도에 제약받지 않았는가? 시장 컨센서스와 지나치게 수렴하지 않았는가? 최종 보고서에는 "정보 풍부도 등급"과 "AI 리서치 한계 선언"을 포함해야 함
8. **정보 희소 시의 정직 원칙** —— 추측으로 프레임워크를 채워 확실성을 위장하느니, 보고서에 "데이터 부족"이라고 공백으로 표기하는 편이 낫다
