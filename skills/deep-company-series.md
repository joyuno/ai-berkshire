# 심층 기업 시리즈: 8편의 장문으로 한 기업을 해부한다

$ARGUMENTS 를 위해 8편의 심층 장문 시리즈를 작성하여 공식계정/영상 계정 등 공개 채널에 발행한다. **핵심 IP는 "잘 쓰는 것"이 아니라 "잘 고치는 것"이다——재무·경제 글의 99%가 본 skill의 사실 검증 기준을 위반하고 있다**.

참고 샘플: `reports/腾讯/《看懂腾讯》/`

---

## 1. 트리거 상황

사용자가 한 기업에 대해 "교과서급" 심층 리서치를 하고, 이를 **시리즈 장문** 형태로 공개 발행하고자 하는 경우. 단일 리서치 보고서와의 차이:
- 8편 약 12만 자, 인식 리셋부터 의사결정 프레임워크까지 완전한 폐루프
- 각 편이 독립적으로 완결(단편 공유에 적합)되지만, 하나의 밸류에이션/경영진/가격 판단 체계를 관통한다
- "한 기업을 이해하는 데 90분을 기꺼이 쓰는" 독자를 위해 쓰며, 증권사 고객을 위한 글이 아니다

**본 skill을 쓰지 않는 상황**: 단일 리서치 보고서, 분기 실적 코멘트, 산업 리서치——그런 것들은 `/investment-research`, `/earnings-review`, `/industry-research` 를 쓴다.

## 2. 시리즈 편목 템플릿(8편)

| # | 편명 템플릿 | 핵심 질문 | 글자 수 |
|---|---------|---------|------|
| 01 | 당신은 X를 이해했다고 생각하지만, 사실 아니다 | 인식 리셋: 흔한 착각 3가지 깨기 | 4,000~5,000 |
| 02 | X의 해자——`<사업 본질 한 문장>` | 해자가 깊은가, 향후 5/10년에도 남아 있을까 | 6,000~8,000 |
| 03 | X의 최대 이익 엔진——`<가장 돈 버는 사업>` | 주력 사업은 무엇이며 왜 지속 가능한가 | 6,000~8,000 |
| 04 | X의 장부에 숨은 또 하나의 회사——`<숨은 자산>` | 투자 포트폴리오 / 자회사 / 숨은 가치 | 8,000~10,000 |
| 05 | AI(또는 당대의 내러티브) 시대, X는 승자인가 패자인가 | 시대 변수: 사업별로 AI 영향 분해 | 8,000~10,000 |
| 06 | 버핏 방식으로 X의 재무제표를 해부한다 | 재무 심화: 매출총이익률/FCF/ROE/SBC | 8,000~10,000 |
| 07 | `<경영진 명언>`——X의 경영진은 맡길 만한가 | 자본 배분 규율 + 정직성 검증 + 후계자 | 8,000~10,000 |
| 08 | 얼마면 살 만하고, 어떤 신호면 반드시 팔아야 하나(시리즈 종장) | DCF 3시나리오 + 레드라인 목록 + 포지션 프레임워크 | 10,000~12,000 |

`00-시리즈 설명.md` 한 편을 목차 인덱스로 추가하되, 발표하지 않는다.

## 3. 글쓰기 스타일 규범

### 어조

- **직접적, 예리함, 군더더기 없음**——첫 문장부터 숫자나 상식을 뒤집는 결론을 준다
- **가치 투자 프레임워크**——버핏/멍거/돤융핑(段永平)/리루 시각을 교차 배치(단, 명언을 쌓아두지 않는다)
- **입장을 예단하지 않는다**——먼저 데이터를 놓고, 다음에 논리를 펴고, 마지막에 결론을 낸다
- **양면을 모두 제시**——모든 핵심 판단에 "그러나 다른 한편으로..."라는 반대편을 붙인다
- **공식계정 체감**——앞 18~20자가 독립적으로 성립해야 한다(모바일 미리보기)

### 금지어

| 금지 | 이유 | 대체 |
|------|------|------|
| 분명히 / 필연적으로 / 반드시 | 주관적 절대화 | 데이터에 따르면 / 증거가 보여주듯 |
| 나는 ~라고 생각한다 / 내 느낌엔 | 주관적 말투 | 삭제하거나 "본 프레임워크에 따르면"으로 변경 |
| 교과서급 / 신의 한 수 | 트래픽 노림 찬사 | 구체적 사실을 기술 |
| 심각하게 불일치 / 심각하게 저평가 | 강한 주관어 | 구체적 할인율 % 제시 |
| 완벽 / 흠잡을 데 없음 | 일방적 판단 | 반대편 관찰을 덧붙임 |

### 제목 스타일

- **대비되는 숫자**나 **반(反)컨센서스 결론**을 후크로 쓴다("15년간 7번 도전 모두 실패", "연봉 4292만, 이익의 0.0017%")
- 부제는 중립적으로 내용을 요약("——`<본질 판단>`")
- **트래픽 노림 비유 회피**: "꼬마 버핏", "중국판 X", "YYDS" 같은 표현은 일절 피한다
- 전문 독자에게 익숙한 용어 사용("버핏"보다 "버크셔", 인명보다 회사명 우선)

## 4. 엄격한 사실 검증 Checklist(핵심 IP)

### 쓰기 전부터 경계해야 할 "가짜 정밀" 함정

1. **확률 가중 기대값**: `30% × A + 50% × B + 20% × C = 기대 +X%` 같은 계산은 거의 다 쓰레기다——확률 배분이 순수 주관이라 독자에게 잘못된 정밀감을 준다. **시나리오 + 트리거 조건 + 방향만 나열하고, 가중 기대값은 계산하지 않는다**.
2. **제3자 추정 MAU/점유율**: QuestMobile/七麦/CBNData 등은 산정 기준 차이가 매우 크다(같은 시점에 2~3배 차이도 난다). **가장 신뢰할 수 있는 두 개의 비교만 앵커로 쓰고, 나머지는 정성적으로 기술한다**.
3. **과거 성장률의 선형 외삽**: `2025년 +33% × 5년 복리 → 2030년 X` 는 금융 문맹식 예측이다. **시나리오 가정 + 고/저 구간 + 약속이 아님**.
4. **비공개 지분율**: 바이트댄스(字节), Halti 류의 비상장 기업 지분은 **공개된 적이 없다**. **구간을 주고 "알 수 없음"이라 표기**한다.
5. **강한 귀인**: 경쟁사 실패 = X 때문. 여러 원인을 모두 나열하고, **본 글은 단일 귀인을 하지 않는다**.

### 수정 시 반드시 돌려야 할 7가지 점검

```
□ 1. 편 간 숫자 일관성: 총 시가총액, Non-IFRS 순이익, 핵심 지분 % 를 시리즈 전체에서 정렬
□ 2. 산정 기준 표기: Non-IFRS / GAAP / Non-IFRS-SBC / FCF 중 무엇을 쓰는지 전문에서 명확히
□ 3. 중복 가산 스캔: 이미 연결된 자회사를 "투자 포트폴리오"에 넣지 않기, SOTP 이중 계상 금지
□ 4. 횡적 비교의 공정성: "주력 사업 PE(현금+포트폴리오 차감)" vs "경쟁사 PE(미차감)" 식 비교 금지
□ 5. 확률 가중 전부 삭제: 위 항목 참조
□ 6. 절대화 표현 전부 약화: grep "显然|必然|严重|教科书|完美"
□ 7. 제3자 데이터 출처 표기: 재무제표가 아닌 데이터마다 "(출처: X)" 부기
```

### 모델 선호

쓰기 전에 **이미 알려진 하드 에러 리스크를 먼저 나열**한다:
- 과거 수익 배수: 반드시 누적 투입 기준 사용(예: Riot 33배, 58배 아님)
- 지분율: 반드시 최신 富途/재무제표 기준 확인(예: 텐센트의 메이퇀 지분 1.5%, 6.4% 아님)
- "배당 분배"의 회계 처리: 처분 간주 이익을 IFRIC 17에 따라 선언일에 인식(예: 징둥은 2021년, 메이퇀은 2022년이나 금액은 작음)
- 총 주식 수는 반등한다: SBC가 연초에 집중 부여되면 주식 수가 단기적으로 상승

## 5. 실행 프로세스

### 단계 1: 조사(01~02편 작성 전 완료)

1. 기업의 최근 5년 연차 보고서, 최신 분기 보고서 읽기
2. 독립 셀사이드 리서치 보고서 최소 3건 읽기(컨센서스 + 반컨센서스 찾기)
3. `/investment-team` 또는 `/investment-research` 로 먼저 내부 리서치 초안 생성
4. 8편의 핵심 논점을 사용자와 확인(다 쓰고 나서 방향이 틀렸음을 발견하는 일 방지)

### 단계 2: 작성(01→08 순서대로, 건너뛰지 않음)

- 각 편을 다 쓰면 먼저 `reports/{회사명}/《看懂{회사명}》/0X-XX.md` 에 저장
- 즉시 GitHub에 푸시하지 않음——사용자 검토 대기
- 사용자가 수정 의견을 주면 수정
- 수정이 끝나야 git push

### 단계 3: 편 간 일관성 스캔(8편 전부 작성 완료 후)

Explore agent를 띄워 8편을 병렬로 스캔하며 다음을 점검:
1. 같은 숫자(시가총액, 순이익, 지분율)가 편 간 일관되는지
2. 같은 용어(FBS, SBC, Non-IFRS)가 처음 등장할 때 설명되는지
3. 참조 관계: 02편에서 "06편 참조"라고 한 것이 실제로 대응하는지
4. 요점 정리 vs 본문의 숫자가 일치하는지

### 단계 4: 발행 전 최종 점검

```bash
# 푸시 전 반드시 로컬에서 한 번 grep(ai-berkshire 프라이버시 규칙에 따라)
grep -r "linxuan\|/Users/\|<사용자 회사 별칭>" reports/ | head
```

이상이 없음을 확인한 뒤에야 `git pull --rebase && git commit && git push`.

## 6. 수정 의견 처리 프로세스

사용자가 수정 의견을 줄 때 다음 순서로 처리한다:

### 1. 먼저 사실을 검증한다(바로 고치지 않는다)

사용자가 "X 데이터가 틀렸다"고 하면, 먼저 Bash/Read로 원본 데이터를 찾아 교차 검증한다:
- ai-berkshire 프로젝트 내 동일 기업의 earnings/재무 보고서 확인
- 富途/공식 공시 확인
- "사용자가 말한 데이터 vs 내가 찾은 데이터 vs 내가 앞서 쓴 데이터" 3자 대조 제시

### 2. 수정 등급 판단

| 등급 | 유형 | 처리 |
|------|------|------|
| 🔥 하드 에러 | 숫자 오류, 귀인 오류, 산정 기준 오류 | 망설임 없이 반드시 수정 |
| ⚠️ 주관화 | 강한 주관어, 절대화, 트래픽 노림 비유 | 약화 또는 삭제 |
| 🔬 세밀도 | 출처 표기, 산정 기준 세분화 | 우선순위 낮음, 가독성과 균형 맞춰 처리 |
| ❓ 신뢰성 부족 | 제3자 추정 편차 큼 | **고치는 것보다 삭제가 더 안전**(사용자 명시 지시) |

### 3. 수정 후 연동 점검

한 곳을 고치면 먼저 "이 숫자/개념을 또 어디서 인용하는가"를 생각한다. 예:
- 총 시가총액 수정 → 시리즈 전체에서 PE / 주력 사업 PE / 할인율 / FCF Yield 연동 수정
- 지분 % 수정 → TOP 10 순위 + 과거 지분 표 + 매도 목록 수정
- 용어 산정 기준 수정 → 최초 정의 + 후속 인용 + 요점 정리 수정

### 4. 푸시 후 즉시 보고

```
푸시 성공(commit hash).
[N]개 수정 요약 [표 포함]:
- 무엇을 고쳤는지
- 연동으로 무엇을 고쳤는지
- 아직 고치지 않은 것은 무엇인지

다음 단계 지시 대기.
```

## 7. 본 skill이 하지 않는 것

- **독자 대신 투자 결정을 내리지 않는다**——모든 편의 말미에 "투자 권유가 아님"
- **주가를 예측하지 않는다**——"시나리오 + 트리거 조건"만 제시
- **"기대 연환산 수익" 가중값을 계산하지 않는다**——주관적 확률 배분은 독자를 오도한다
- **"X 거물도 보유 중"이라고 쓰지 않는다**——타인의 보유로 자기 판단을 뒷받침하는 것은 가치 투자에 반한다
- **8편을 무리하게 다 쓰지 않는다**——어떤 편에 충분한 독립 내용이 없으면(예: 어떤 기업의 경영진이 특별하지 않다면) 다른 편에 통합하거나 편수를 줄인다

## 8. 컴플라이언스와 프라이버시

- 모든 공개 보고서는 **공개 정보만** 사용(재무제표, 공식 공시, 증권사 리서치, 저명한 제3자 기관)
- 어떤 **사용자 개인 정보**도 사용하지 않음(회사 별칭, 내부 IM, 비공개 보유 정보)
- 푸시 전에 반드시 grep으로 `linxuan` / `/Users/` / 사용자 회사 별칭 등 프라이버시 필드를 스캔(`~/.claude/projects/-Users-linxuan/memory/feedback_privacy_upload.md` 참조)
- 공개 서명은 사용자의 다층 신원 전략에 따르며, 혼용하지 않는다

## 한 문장 요약

**《X 이해하기 시리즈》를 쓰는 핵심 역량 ≠ 잘 쓰는 것, 엄격하게 고치는 것이다**——
재무·경제 장문의 89%는 가짜 정밀 숫자, 주관적 가중 기대값, 절대화 표현으로 죽는다. 본 skill의 존재 이유는 바로 이 함정들을 전부 표시해 두고, 쓰기 전에 피하고, 쓴 뒤에 깨끗이 쓸어내는 것이다.
